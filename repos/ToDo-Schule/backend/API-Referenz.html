Kollaboratives ToDo-Backend für die PWA.

Vollständige REST-API mit Authentifizierung, Teams, Share-Links, Kommentaren, Audit-Trail und Echtzeit-Synchronisation über WebSockets — umgesetzt in reinem PHP & SQL, ohne externe Bibliotheken.

Authentifizierung & Benutzer

JWT mit kurzlebigem Access-Token und rotierendem Refresh-Token. Passwörter via bcrypt.

POST /api/auth/register Konto anlegen · rate-limited

Request

{
  "email": "lea@schule.de",
  "password": "mind8Zeichen",
  "name": "Lea M."
}

Response · 201

{
  "user": { "id": 1, "email": "lea@schule.de", "name": "Lea M." },
  "accessToken": "eyJhbG...",
  "refreshToken": "eyJhbG...",
  "expiresIn": 900
}
POST /api/auth/login Token-Paar erhalten · rate-limited

Request

{ "email": "lea@schule.de", "password": "mind8Zeichen" }

Response · 200

{ "user": { … }, "accessToken": "…", "refreshToken": "…", "expiresIn": 900 }
POST /api/auth/refresh Rotation: altes Token wird widerrufen

Request

{ "refreshToken": "eyJhbG..." }

Response · 200

{ "accessToken": "…neu…", "refreshToken": "…neu…", "expiresIn": 900 }
POST /api/auth/logout Refresh-Token serverseitig invalidieren

WebSocket-Anbindung

Nativer Browser-WebSocket, kein socket.io. Der Access-Token wird im Handshake übergeben.

WSws://localhost:8090/?token=<accessToken>

Client (PWA)

// JWT im Handshake — Äquivalent zu socket.io auth.token
const ws = new WebSocket(`ws://localhost:8090/?token=${token}`);

ws.onmessage = (e) => {
  const { event, payload } = JSON.parse(e.data);
  // task:created | task:updated | task:deleted
  // comment:added | user:assigned | team:member_added
  store.apply(event, payload);
};

Nach Verbindung

Socket tritt automatisch bei:
  user:<id>            // persönlicher Room
  team:<id> …          // je Teammitgliedschaft
